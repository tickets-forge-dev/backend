# Story 8.4 - BMAD Migration Plan & Architecture

## The Core Challenge

**BMAD is conversational. Our platform is NOT a chat.**

BMAD expects:
- Multiple back-and-forth exchanges
- User answering questions inline
- Checkpoint approvals after each section

Our platform has:
- Form-based input
- Ticket detail view
- Generation progress UI

**Solution: Multi-Stage Wizard with Review Points**

---

## New UX Flow: Staged Generation

## Summary

| Aspect | BMAD (Chat) | Our Platform (Forms) |
|--------|-------------|---------------------|
| Context confirmation | "Does this look right?" | Stage 2 review screen |
| Questions | Back-and-forth chat | Form with radio/checkbox |
| Draft review | "Here's what I generated" | Stage 3 with edit cards |
| Approval | "Ready to proceed?" | Stage 4 confirmation |
| Suspension points | Natural conversation | API calls between stages |

**The key insight:** We replace BMAD's conversational flow with a **multi-stage wizard** where each stage is a review/edit point. The user experience is actually **better** because:

1. All context visible at once (not scrolling through chat)
2. Questions presented as forms (easier than typing)
3. Editable cards (inline editing vs "can you change X")
4. Clear progress (stage 2/4 vs unknown conversation length)
